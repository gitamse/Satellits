# Правила генерации лендингов

Ты делаешь статическую верстку на основе Figma-макета и существующей базы.

## Структура файлов

Каждый лендинг состоит из:
- index.html
- дополнительные страницы: page-name.html
- css/main.css
- css/media.css
- js/main.js
- images/

Не использовать React, Vue, Tailwind, Bootstrap.

## Главный принцип

Не придумывать новую архитектуру, если похожий блок уже есть в satellits/.

Сначала:
1. Проанализируй Figma-макет.
2. Найди похожие реализации в satellits/.
3. Составь карту соответствий:
   - блок Figma → существующий пример → классы → CSS.
4. Только потом создавай новую верстку.

## Имена классов

Использовать существующий стиль классов из базы.
Если в базе используется БЭМ — продолжать БЭМ.
Новые классы вводить только если нет подходящего аналога.

## CSS

main.css:
- базовые стили
- desktop
- компоненты

media.css:
- только адаптив
- сохранять существующие breakpoint-подходы из базы

## JS

main.js:
- все скрипты уже встречаются в базе. Нужно либо добавить необходимые, либо удалить не нужные

## Перед финалом

Проверить:
- все изображения подключены
- нет битых путей
- адаптив desktop/tablet/mobile
- классы не конфликтуют
- результат похож на Figma
